# README: Notification & Warehouse Service

Это комплексный демонстрационный проект на Go, реализующий современные паттерны разработки backend-систем. Проект объединяет в себе несколько бизнес-модулей: от обработки изображений через Kafka до сложной финансовой аналитики и ролевой модели доступа на триггерах БД.

---

## 🚀 Основные возможности

### 1. Сервис уведомлений (RabbitMQ)
* Асинхронная отправка уведомлений.
* Планировщик задач для отложенных уведомлений.

### 2. Система бронирования (PostgreSQL Transactions + Cron)
* Атомарное бронирование мест на мероприятия.
* Фоновый воркер (Cron) для автоматической отмены неоплаченных броней по истечении TTL (Time To Live).

### 3. Sales Tracker (SQL Analytics)
* Расчет сложных аналитических метрик прямо в БД: медиана, 90-й перцентиль, средний чек.
* Выгрузка всех транзакций в формате CSV одним кликом.

### 4. Warehouse Control (RBAC + DB Auditing)
* Ролевая модель доступа: `admin`, `manager`, `viewer`.
* Полный аудит изменений товаров реализован через триггеры в PostgreSQL (антипаттерн в учебных целях).
* Сохранение детальной истории изменений: кто именно, когда и какие данные изменил (`old_data` / `new_data`).

### 5. Обработка изображений (Kafka)
* Асинхронная обработка тяжелых задач (ресайз, фильтры) через брокер сообщений Kafka.

## ⚙️ Настройка и запуск

### 1. Клонирование и зависимости

'''git clone <your-repo-url>
cd notification-service
go mod tidy'''

### 2. Конфигурация
Проект использует систему разделения секретов. **Никогда не пушьте `config.yaml` в репозиторий!**

1. Скопируйте шаблон:
   ```bash
   cp configs/config.example.yaml configs/config.yaml
   ```

2. Откройте `configs/config.yaml` и заполните его своими данными:
   * DSN для PostgreSQL.
   * Токен Telegram бота и Chat ID.
   * Адреса брокеров (RabbitMQ/Kafka).

### 3. Запуск инфраструктуры
Для работы всех систем необходимо поднять Docker-контейнеры:
```bash
docker-compose up -d
```

### 4. Запуск приложения
```bash
go run cmd/app/main.go --config=configs/config.yaml
```

## 🔒 Безопасность

* **`.gitignore`**: Файл настроен на игнорирование бинарных файлов, папок с загрузками и конфига с секретами.
* **RBAC**: Доступ к складским операциям защищен Middleware, проверяющим роль в JWT-токене.
* **DB Context**: Имя пользователя пробрасывается в триггеры через сессионные переменные PostgreSQL (`myapp.current_user`).
